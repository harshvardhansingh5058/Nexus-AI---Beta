Quick Start

Get up and running in minutes. Register your app, grab your Client ID, and make your first request.

1. Create a developer account

Sign up at the developer portal to access your dashboard. Once your account is verified you can register applications and generate access credentials.

2. Register your application

Each application gets a unique client_id and client_secret. The client_id is used for public requests; the client_secret is used for OAuth flows only.

3. Make your first request

bashcurl -H "Authorization: Client-ID YOUR_CLIENT_ID" \ "https://api.yourapp.com/v1/photos"

Keep your client_secret private. Never expose it in client-side code or public repositories.

Authorization

The API supports public authentication for read-only access and OAuth 2.0 for user-specific actions.

Public Authentication

For read-only endpoints, pass your client_id via the HTTP Authorization header.

httpAuthorization: Client-ID YOUR_CLIENT_ID

User Authentication (OAuth 2.0)

To act on behalf of a user, redirect them to the authorization URL. After approval you'll receive a code to exchange for an access token.

httpGET https://yourapp.com/oauth/authorize ?client_id=YOUR_CLIENT_ID &redirect_uri=YOUR_REDIRECT_URI &response_type=code &scope=public+write_likes

Access tokens expire after 1 hour. Use the refresh token to silently obtain a new one without re-prompting the user.

Location

All API requests are made to the following base URL. Requests must be made over HTTPS.

https://api.yourapp.com

Prefix all endpoint paths with /v1. For example, GET /v1/photos.

Version

The current stable version of the API is v1, specified in the URL path.

When a breaking change is introduced, a new version will be released alongside a migration guide. Old versions are supported for a minimum of 6 months after deprecation is announced.

HTTP Verbs

The API uses standard HTTP verbs to indicate the type of action being performed.

VerbDescription
GETUsed for retrieving resources.
POSTUsed for creating resources or performing actions.
PUTUsed for replacing resources or collections.
DELETEUsed for deleting resources.

Error Messages

The API uses conventional HTTP response codes. Errors return a JSON body with a human-readable message.

json{ "errors": ["OAuth error: The access token is invalid"] }
CodeMeaning
200OK — Request succeeded.
400Bad Request — Missing or invalid parameters.
401Unauthorized — Missing or invalid credentials.
403Forbidden — Access denied to this resource.
404Not Found — The resource doesn't exist.
429Too Many Requests — Rate limit exceeded.
500Internal Server Error — Something went wrong on our end.

Search Photos

Search for photos by keyword. Returns a paginated list of results matching your query and applied filters.

GET/v1/search/photos

Parameters

ParamTypeDescription
querystringRequiredSearch terms.
pageintegerOptionalPage number to retrieve. (default: 1)
per_pageintegerOptionalNumber of items per page. (default: 10, max: 30)
orientationstringOptionalFilter by orientation: landscape, portrait, squarish
colorstringOptionalFilter by color: black_and_white, red, blue, etc.

Example Request

bashcurl -H "Authorization: Client-ID YOUR_CLIENT_ID" \ "https://api.yourapp.com/v1/search/photos?query=mountains&per_page=5"

Get a Random Photo

Retrieve one or more random photos. Use optional parameters to narrow the pool.

GET/v1/photos/random

Parameters

ParamTypeDescription
querystringOptionalLimit to photos matching this search term.
orientationstringOptionalFilter by orientation.
collectionsstringOptionalComma-separated collection IDs to filter by.
countintegerOptionalNumber of photos to return. (max: 30)

When count is greater than 1, the response is an array of photo objects rather than a single object.

List New Photos

Get a paginated list of the latest photos. Updated daily.

Parameters

ParamTypeDescription

Pagination Headers

Paginated responses include standard HTTP link headers so you can navigate between pages programmatically.

HeaderDescription
X-TotalTotal number of items across all pages.
X-Per-PageNumber of items returned per page.
X-PageCurrent page number.
LinkContains prev, next, first, and last page URLs.

Rate Limiting

The API rate limits requests to ensure fair usage. Current limits are shown in the response headers of every request.

PlanRequests / hourNotes
Demo50Development & testing only. Not for production use.
Standard5,000Default after app approval.
ProductionCustomApply after launching publicly.

When exceeded, the API returns 429 Too Many Requests. Check the X-Ratelimit-Reset header for when your limits reset.

Rate Limit Headers

HeaderDescription
X-Ratelimit-LimitMaximum requests allowed per hour.
X-Ratelimit-RemainingRequests remaining in the current window.
X-Ratelimit-ResetUnix timestamp when the limit resets.

Guidelines & Crediting

Follow these guidelines in your production application to keep your API access in good standing.

Attribution

You must credit the photographer and Nexus wherever photos appear in your application. This includes displaying the photographer's name and linking back to their profile page.

Hotlinking

You may hotlink images directly from the CDN URLs returned by the API. Do not re-host or redistribute photos in bulk outside of standard caching.

Prohibited Uses

You may not use the API to create a service that directly competes with Nexus, sell or redistribute access to the API, or use photos in ways that violate photographer rights.

Apps that do not follow attribution guidelines may have their API access suspended after a warning is issued.

Deprecation Policy

We provide at least 6 months' notice before removing or making breaking changes to any API endpoint.

How We Notify You

Deprecated endpoints return a Deprecation response header containing the planned removal date. We also post announcements to the developer changelog and send email notifications to affected application owners.

Current Deprecations

No active deprecations at this time.
